# 上海文化广场余票监测系统 — 使用教程

## 简介

本系统用于监测上海文化广场（shcstheatre.com）的演出票务状态，支持多用户独立使用、自动刷新监测余票、微信推送通知。

**访问地址**：`https://ticket-60.site`

> 详细使用说明请参考：
> - [用户使用手册](USAGE_USER.md) — 注册、登录、监测、推送配置
> - [管理员手册](USAGE_ADMIN.md) — 开发者后台、用户管理、服务器配置

---

## 快速开始

### 1. 注册账号

1. 打开 `https://ticket-60.site`
2. 点击 **注册** 标签
3. 输入用户名和密码
4. 点击 **注册并进入**

### 2. 登录剧场账号

1. 输入上海文化广场手机号和密码
2. 点击获取验证码（系统自动 OCR 识别）
3. 点击登录

### 3. 选择演出并监测

1. 进入 **选择演出** → 加载剧目 → 选择场次和票档
2. 进入 **余票监测** → 设置间隔和数量 → 开始监测
3. 有票时自动下单，成功后推送通知

## 功能特性

| 功能 | 说明 |
|------|------|
| 多用户隔离 | 每个用户独立的监测进程、日志、推送配置 |
| 自动 OCR 验证码 | 使用 ddddocr 自动识别验证码 |
| 自动选座下单 | 支持选座和非选座两种模式 |
| 微信推送 | 支持 WxPusher、Server酱、PushPlus |
| 配置持久化 | 推送配置自动保存，重启不丢失 |
| 管理员后台 | 用户管理、监测活动查看、远程操作 |

## 技术架构

```
用户浏览器 ──HTTPS──→ Nginx(443) ──HTTP──→ Flask(5000)
                        │
                        ├── Playwright (无头浏览器)
                        ├── ddddocr (验证码 OCR)
                        ├── requests (HTTP 请求)
                        └── SSE (实时日志推送)
```

- **前端**：纯 HTML/CSS/JS 单页应用
- **后端**：Python Flask，运行在 Linux 服务器
- **实时通信**：SSE（Server-Sent Events）推送日志
- **反向代理**：Nginx + Let's Encrypt HTTPS
