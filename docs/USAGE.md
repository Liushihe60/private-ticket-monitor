# 上海文化广场余票监测系统 — 使用教程

## 一、系统简介

本系统用于监测上海文化广场（shcstheatre.com）的演出票务状态，支持自动刷新监测余票、微信推送通知。

**访问地址**：`http://49.235.110.106:5000`

---

## 二、电脑端使用教程

### 2.1 打开网页

1. 打开浏览器（推荐 Chrome / Edge）
2. 地址栏输入 `http://49.235.110.106:5000`
3. 进入深色主题的操作界面，左侧为导航栏

### 2.2 登录账号

1. 点击左侧 **「账号登录」**
2. 输入你的上海文化广场 **手机号** 和 **密码**
3. 点击 **「点击获取」** 按钮，系统会自动获取验证码图片并识别
4. 验证码图片会显示在输入框右侧，OCR 识别结果会自动填入
5. **核对验证码**：如果识别不准确，手动修改输入框中的数字
6. 点击 **「登 录」** 按钮
7. 显示「登录成功」即可进入下一步

### 2.3 选择演出

1. 点击左侧 **「选择演出」**
2. 点击 **「加载可购票剧目」** 按钮
3. 左侧列表出现可选剧目，**点击选择**你要监测的剧目
4. 中间列表自动加载该剧目的场次，显示信息包括：
   - 日期时间
   - 状态（可购 / 未开售 / 已结束）
   - 类型（选座 / 无座）
   - 剩余票数
5. **点击选择**目标场次
6. 右侧列表自动加载票档，显示价格和状态（有票 / 已售罄）
7. **点击选择**你想要的票档
8. 下方会显示当前选择摘要，如：`剧名 → 2026-06-15 19:30 → ¥680`

### 2.4 开始监测

1. 点击左侧 **「余票监测」**
2. 设置 **刷新间隔**（默认 3 秒，建议 2-5 秒）
3. 设置 **购买数量**（1-4 张）
4. 确认下方显示的选择信息无误
5. 点击 **「开始监测」**
6. 状态指示灯变为绿色呼吸灯，日志区实时显示监测信息
7. 当检测到有票时，系统会自动尝试下单
8. 下单成功后会通过微信推送通知（需先配置推送）

### 2.5 停止监测

点击 **「停止」** 按钮即可停止监测。

### 2.6 配置微信推送

1. 点击左侧 **「微信推送」**
2. 选择推送方式（推荐 WxPusher）
3. 填入对应的 Token / Key
4. 如选择 WxPusher，还需填入你的 UID
5. 点击 **「保存配置」**
6. 点击 **「发送测试」** 验证是否能收到微信消息

## 三、手机端使用教程

### 3.1 打开网页

1. 打开手机浏览器（Safari / Chrome / 微信内置浏览器均可）
2. 地址栏输入 `http://49.235.110.106:5000`
3. 页面会自动适配手机屏幕，导航栏变为顶部横向排列

### 3.2 操作流程

手机端操作与电脑端完全一致，界面会自动调整为单列布局：

1. **登录**：输入手机号、密码 → 获取验证码 → 登录
2. **选择演出**：加载剧目 → 依次选择剧目、场次、票档
3. **开始监测**：设置间隔和数量 → 开始监测
4. **推送配置**：配置微信推送 token

### 3.3 手机端小技巧

- 可以将网页 **添加到主屏幕**，像 App 一样使用：
  - **iPhone**：Safari 打开 → 点底部分享按钮 → 「添加到主屏幕」
  - **Android**：Chrome 打开 → 点菜单 → 「添加到主屏幕」
- 日志区域可以上下滑动查看历史记录
- 监测过程中不要关闭浏览器标签页（后台运行不影响，但关闭标签页会断开 SSE 日志连接，不影响监测本身）

## 四、技术架构说明

### 4.1 整体架构

```
用户浏览器 ──HTTP──→ 服务器(Flask:5000) ──→ 上海文化广场API
                        │
                        ├── Playwright (Chromium 无头浏览器)
                        ├── ddddocr (验证码 OCR)
                        ├── requests (HTTP 请求)
                        └── SSE (实时日志推送)
```

- **前端**：纯 HTML/CSS/JS 单页应用，由 Flask 直接托管
- **后端**：Python Flask，运行在 Linux 服务器上
- **实时通信**：SSE（Server-Sent Events）推送日志到浏览器

### 4.2 登录与身份验证流程

```
用户输入 手机号+密码
        │
        ▼
点击「获取验证码」
        │
        ▼
服务器启动 Playwright 无头 Chromium ──→ 访问文化广场登录页
        │                                    │
        │                              截图验证码元素(#yanzhengma)
        │                                    │
        ▼                                    ▼
  ddddocr OCR 识别 ──→ 返回验证码图片(base64) + 识别文本
        │
        ▼
浏览器显示验证码图片，自动填入 OCR 结果
用户可手动修正
        │
        ▼
点击「登录」──→ 服务器用 requests 库发送 HTTP POST 请求
        │         到文化广场 WebAPI，携带：
        │         - 手机号、密码
        │         - 验证码文本
        │         - Playwright 获取的 cookies
        │
        ▼
返回 token（身份凭证），存储在服务器内存中
        │
        ▼
后续所有请求（剧目列表、场次、票档、下单）均携带此 token
```

**关键点**：
- 验证码获取使用 Playwright（模拟浏览器），因为验证码图片是动态生成的
- 登录请求使用 requests（纯 HTTP），不需要浏览器
- Cookie 在 Playwright 获取验证码时一并获得，传递给 requests 保持会话
- Token 存储在服务器内存，重启服务需要重新登录

### 4.3 监测流程

```
循环（每 N 秒）：
  ├── 查询票档状态（HTTP API）
  ├── 判断是否有票（sold_out==0 且 seat_cnt>0）
  │     ├── 无票 → 记录日志，继续循环
  │     └── 有票 → 尝试下单
  │           ├── 选座模式：Playwright 打开座位图，自动选座
  │           └── 非选座模式：直接调用下单 API
  │           ├── 下单成功 → 微信推送通知，停止监测
  │           └── 下单失败 → 记录日志，继续循环
  └── 错误处理：连续失败超过阈值时记录并刷新
```

## 五、安全风险分析

### 5.1 当前风险（TCP 5000 全开放）

**风险等级：中高**

| 风险项 | 严重程度 | 说明 |
|--------|---------|------|
| 无身份认证 | **高** | 任何人知道 IP 就能访问，可以操作你的登录状态、启动/停止监测 |
| CORS 全开放 | **中** | `Access-Control-Allow-Origin: *` 允许任意网站调用 API |
| HTTP 明文传输 | **中** | 手机号、密码在网络上明文传输，可被中间人截获 |
| 无 HTTPS | **中** | 浏览器不显示安全锁标志，容易被钓鱼网站仿冒 |
| 端口暴露 | **低** | 5000 端口对外开放，可能被扫描器探测到 |
| Flask debug 模式 | **无** | 当前 debug=False，这点是安全的 |

**可能的攻击方式**：
1. **未授权访问**：任何人访问 `http://49.235.110.106:5000` 就能操作你的监测任务
2. **凭证泄露**：HTTP 明文传输手机号和密码，同一网络下的攻击者可以嗅探到
3. **会话劫持**：攻击者可以通过 API 获取你的文化广场 token
4. **资源滥用**：攻击者可以反复调用验证码接口，消耗服务器 CPU/内存（Playwright 启动浏览器很耗资源）

**实际威胁评估**：
- 如果只是个人使用且不告诉别人地址，风险较低（IP 地址不容易被猜到）
- 但如果在公共网络（如公司 WiFi）使用，密码可能被嗅探
- 服务器本身不会被「攻破」，因为没有开放 SSH 密码登录以外的入口

### 5.2 域名 + 反向代理 + HTTPS 的安全提升

**配置方案**：

```
用户 ──HTTPS──→ Nginx(443) ──HTTP──→ Flask(5000, 仅本地)
                  │
                  ├── SSL/TLS 证书（Let's Encrypt 免费）
                  ├── 域名验证
                  └── 可选：HTTP Basic Auth / IP 白名单
```

**安全提升对比**：

| 项目 | 当前（裸 IP+HTTP） | 配置后（域名+HTTPS） |
|------|-------------------|---------------------|
| 传输加密 | 无 | TLS 1.3 加密 |
| 身份验证 | 无 | 可加 HTTP Basic Auth |
| 密码保护 | 明文传输 | 加密传输 |
| CORS 限制 | `*`（全开放） | 可限制为特定域名 |
| 端口暴露 | 5000 对外 | 仅 443 对外，5000 仅本地 |
| SEO/品牌 | IP 地址 | 自定义域名 |
| 浏览器警告 | 无（但无安全锁） | 显示安全锁标志 |

**难度评估**：

| 步骤 | 难度 | 耗时 |
|------|------|------|
| 购买域名 | 简单 | 10 分钟 |
| DNS 解析配置 | 简单 | 5 分钟 |
| 安装 Nginx | 简单 | 5 分钟 |
| 申请 SSL 证书（Let's Encrypt） | 中等 | 15 分钟 |
| 配置 Nginx 反向代理 | 中等 | 10 分钟 |
| 配置自动续期证书 | 简单 | 5 分钟 |

**总计**：约 50 分钟，需要有域名（年费约 50-100 元）。

**推荐操作**：
1. 买一个便宜的域名（如 `.xyz` 首年 8 元）
2. DNS A 记录指向 `49.235.110.106`
3. 服务器安装 Nginx + Certbot
4. 配置反向代理，Flask 绑定 `127.0.0.1:5000`（不再对外暴露）
5. 启用 HTTPS

这样既安全又好看（如 `https://ticket.你的域名.com`）。
